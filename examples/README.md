# Examples

These examples show how corpus pattern lessons become practical prompt artifacts. Each example should be small enough to copy, test, and adapt without reading the whole corpus.

Future examples should include these fields:

- **Use case**: The real task the prompt supports.
- **Applied pattern(s)**: Pattern lesson names from `prompt-corpus/PATTERN_LESSONS_INDEX.md`.
- **Source pattern reference**: The index section name, or source entry only when the exact entry is known.
- **Final prompt**: The usable prompt artifact.
- **Why this structure**: A short explanation of the control points used.
- **Failure / fallback rule**: What the assistant should do when information is missing, stale, ambiguous, or unsupported.
- **Test input**: A realistic input to try.
- **Expected behavior**: What a good response should do and avoid.

Do not invent corpus entry IDs. If the exact source entry is uncertain, refer to the pattern section name from the index.
